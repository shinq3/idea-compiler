# 機能提案

## 機能一覧 (Must / Should / Could)

### Must
- **音声入力による会議サマリー作成機能**  
  - 目的: 音声入力をもとに、要点、決定事項、論点、次アクションを含む包括的な会議サマリーを自動生成する。  
  - 対象ユーザー: 会議参加者、プロジェクトマネージャー、タイムリーな議事録が必要なチームメンバー。  
  - 根拠: Input #4 要件；プロジェクト概要「スコープ」。

- **プロジェクト発足時の資料作成機能**  
  - 目的: プロジェクト発足時にキックオフ資料および機能提案書を自動生成する。  
  - 対象ユーザー: プロジェクトリード、ステークホルダー、PMO。  
  - 根拠: Input #4 要件；プロジェクト概要「スコープ」。

- **RAGの導入**  
  - 目的: RAG（Retrieval-Augmented Generation）を用いて、複数回の会議サマリー統合・要約機能を実現する。  
  - 対象ユーザー: ナレッジマネージャー、複数セッションのデータを統合するアナリスト。  
  - 根拠: Input #7 要件。

- **会議サマリーの統合**  
  - 目的: 複数回の会議サマリーをひとつの整合的なドキュメントに統合する。  
  - 対象ユーザー: 定例会議やフェーズをまたぐ会議を行うチーム。  
  - 根拠: Input #7 要件。

- **ローカルバックアップ録音機能**  
  - 目的: 音声取得失敗時に備え、マイク本体で録音データをローカル保存し、データロスを防止する。  
  - 対象ユーザー: 会議参加者全員およびデータ整合性を担保する管理者。  
  - 根拠: Input #5 要件；リスク「音声キャプチャ失敗」(Input #5)。

### Should
- **リアルタイムAPIによるライブ字幕表示**  
  - 目的: 会議中にリアルタイムで字幕を表示し、即時の内容把握とアクセシビリティを向上させる。  
  - 対象ユーザー: リモート参加者、聴覚障がい者、ファシリテーター。  
  - 根拠: Input #2 議論；プロジェクト概要「スコープ」。

- **文字起こし方法の選択**  
  - 目的: Whisperベースのアップロード方式とリアルタイムAPI方式を選択できるようにし、信頼性と即時性をトレードオフできる。  
  - 対象ユーザー: IT管理者、プロダクトオーナー。  
  - 根拠: Input #5 決定；制約「リアルタイムAPIのコスト」(Input #5)。

### Could
- **外部連携機能（オプション）**  
  - 目的: 第2フェーズでECサイトやチケット購入サイトとの連携を追加検討する。  
  - 対象ユーザー: ビジネス開発チーム、サードパーティ連携を検討する担当者。  
  - 根拠: Input #8 要件。

- **会議サマリーエクスポート**  
  - 目的: PDF、Docx、Markdown形式でサマリーを出力し、共有・アーカイブを容易にする。  
  - 対象ユーザー: ドキュメント担当者、コンプライアンス部門。  
  - 根拠: プロジェクト概要の機能候補。

- **サマリー編集・再生成機能**  
  - 目的: ユーザーのプロンプトに応じた手動編集とAIによる再生成を可能にする。  
  - 対象ユーザー: 会議オーガナイザー、内容レビュアー。  
  - 根拠: プロジェクト概要の機能候補。

## 依存関係
- Whisperベースの文字起こしエンジン (アップロード)  
- リアルタイム音声認識API (コスト承認待ち) [Input #5]  
- マイク本体のローカルストレージ機能 [Input #5]  
- RAGインフラおよびインデックス保存（クラウド/オンプレミス検討中） [未解決項目]  
- キックオフ資料・提案書のテンプレートライブラリ  
- 認証・セキュリティ基盤  

## 受け入れ基準
1. 音声アップロード後1分以内に要点・決定事項・次アクションを含むサマリーを生成できること。  
2. キックオフ資料・機能提案書は定義されたテンプレートに則り、章構成も正しく生成されること。  
3. すべての会議でローカル録音バックアップが作成され、取得失敗時に復元できること。  
4. RAGによる統合で3回分以上の会議サマリーを一文書にまとめ、文脈を保持していること。  
5. MVP段階でのライブ字幕表示は制御下の環境で90%以上の認識精度を達成すること。  

## 除外項目
- Zoom/Teams等との会議ツール連携 (第2フェーズ) [プロジェクト概要 “スコープ”]  
- 初期フェーズでのEC/チケットサイト連携 [Input #8]  
- 承認ワークフロー、権限管理、監査ログ [プロジェクト概要 “スコープ”]  
- テンプレート詳細要件策定・運用体制の整備 [プロジェクト概要 “スコープ”]
