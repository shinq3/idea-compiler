

WeCALL相当システム 機能詳細（バイブコーディング向け）

1. システム概要

イベント運営向けのオールインワンプラットフォームとして、以下の3系統を中心に構成する。
	1.	イベント予約 / 個人情報取得 / チェックイン
	2.	整理券発券 / 呼び出し管理
	3.	イベント物販注文 / モバイルオーダー / 受け渡し管理

さらに、LINEまたはWebから利用でき、通知・管理画面・来場データ活用まで含む前提で設計する。  ￼

⸻

2. 想定ユーザーロール

2-1. 来場者
	•	予約ページにアクセスする
	•	必要情報を入力する
	•	予約完了情報を確認する
	•	通知を受け取る
	•	当日チェックインする
	•	整理券を取得する
	•	呼び出し通知を受ける
	•	物販商品を注文する
	•	受け渡し通知を受ける

2-2. 現場スタッフ
	•	予約一覧を確認する
	•	来場者のチェックイン処理を行う
	•	整理券番号を呼び出す
	•	注文内容を確認する
	•	商品受け渡し完了を登録する

2-3. イベント管理者
	•	イベントを作成・編集する
	•	フォーム項目を設定する
	•	通知文面を設定する
	•	予約・来場・呼び出し・注文実績を確認する
	•	各機能を単独または組み合わせで有効化する

	•	会場名
	•	会場住所
	•	公開状態
	•	利用機能フラグ
	•	予約利用

	•	物販利用
	•	通知設定

	•	イベント作成 / 編集
	•	公開プレビュー
	•	イベント複製
	•	公開停止

5. 予約 / 個人情報 / チェックイン機能 詳細

元ページでは、予約・個人情報取得・チェックインは独立導入可能で、LINEやWeb経由で利用できるとされている。体験フローは、予約ページアクセス→フォーム入力→予約完了→前日通知→チェックイン。  ￼

5-1. 予約受付機能

目的

来場前に参加枠を予約させる。

ユーザーフロー
	1.	予約ページへアクセス
	2.	予約フォーム入力
	3.	確認
	4.	予約確定
	5.	完了画面表示
	6.	通知送信

取得項目（デフォルト）
	•	氏名
	•	フリガナ
	•	電話番号
	•	メールアドレス
	•	LINE識別子
	•	来場予定日時 / 枠
	•	参加人数
	•	備考
	•	同意チェック

可変フォーム項目
	•	テキスト
	•	ラジオ
	•	チェックボックス
	•	セレクト
	•	日付
	•	数値
	•	必須 / 任意フラグ

バリデーション
	•	必須チェック
	•	メール形式
	•	電話番号形式
	•	重複予約制御
	•	定員超過制御
	•	予約締切判定

予約状態
	•	仮受付
	•	予約確定
	•	キャンセル
	•	来場済
	•	無断未訪問

管理画面機能
	•	予約一覧
	•	条件検索
	•	ステータス変更
	•	手動チェックイン
	•	予約詳細確認
	•	個人情報閲覧

⸻

5-2. 個人情報取得機能

目的

イベントに必要な来場者情報を取得し、運営データとして活用する。

要件
	•	イベントごとに取得項目を設定できる
	•	必須 / 任意を切り替え可能
	•	同意取得履歴を保持する
	•	LINEユーザーとの紐付けが可能
	•	管理画面から一覧 / 詳細確認が可能

保存データ
	•	participant_id
	•	event_id
	•	profile fields JSON
	•	consent_version
	•	consent_at
	•	source_type（LINE / Web）
	•	created_at
	•	updated_at

注意点
	•	個人情報項目は固定カラムより JSON + 定義テーブルの方が拡張しやすい
	•	同意文バージョンは別管理した方が後で地雷を踏みにくい

⸻

5-3. チェックイン機能

目的

当日の来場確認をスムーズに行う。

入力方式
	•	予約番号入力
	•	QRコード読み取り
	•	氏名検索
	•	整理券番号検索

処理内容
	•	対象予約検索
	•	来場資格確認
	•	チェックイン実行
	•	時刻記録
	•	スタッフ記録
	•	重複チェックイン防止

エラーケース
	•	予約なし
	•	キャンセル済
	•	受付対象外時間
	•	すでにチェックイン済
	•	整理券未呼び出し

管理画面
	•	チェックイン専用画面
	•	一覧から即時チェックイン
	•	当日来場状況リアルタイム表示

6. 整理券機能 詳細

元ページでは、整理券機能はLINE上で順番受付を行い、呼び出しや待ち状況がリアルタイム反映され、QRコードなどから発券ページへアクセスし、管理画面から呼び出し、利用者へ通知し、番号確認後にチェックインする流れが示されている。  ￼

6-1. 整理券発券機能

目的

混雑や行列を減らし、待機列をデジタル管理する。

ユーザーフロー
	1.	QRコードまたはURLから発券ページにアクセス
	2.	必要事項入力
	3.	整理券発行
	4.	発券番号表示
	5.	待機状況確認

発券時取得項目
	•	氏名またはニックネーム
	•	人数
	•	LINE識別子または連絡先
	•	希望カテゴリ（任意）
	•	注意事項同意

発番仕様
	•	イベント単位連番
	•	枠単位連番（必要なら）
	•	プレフィックス付与対応
	•	呼出順序の手動調整可

整理券状態
	•	発券済
	•	待機中
	•	呼出中
	•	呼出済
	•	来場済
	•	失効
	•	キャンセル

表示項目
	•	自分の番号
	•	現在呼出中番号
	•	待ち人数
	•	予想待ち時間（拡張）
	•	注意事項

⸻

6-2. 呼び出し管理機能

目的

スタッフが管理画面から待機者を順番に呼び出す。

管理画面要件
	•	待機一覧表示
	•	次番号呼出ボタン
	•	個別呼出
	•	再呼出
	•	スキップ
	•	呼出取消
	•	来場登録
	•	不在処理

通知要件
	•	呼び出し時にLINE通知
	•	通知失敗時の再送管理
	•	呼出後の有効時間表示

リアルタイム要件
	•	待機一覧の自動更新
	•	来場者画面側もリアルタイム更新
	•	多端末同時操作の競合制御

競合対策
	•	同一整理券を複数スタッフが同時呼出しできない
	•	楽観ロックまたは version カラムを使用

7. イベント物販注文機能 詳細

7-1. 商品注文受付機能

	4.	サイズ / カラー選択
	5.	数量入力
	6.	注文確認
	7.	注文確定
	8.	完了画面表示

商品マスタ項目
	•	商品ID
	•	商品名
	•	商品説明
	•	商品画像
	•	税込価格
	•	在庫管理有無
	•	公開状態
	•	表示順
	•	販売期間
	•	受取区分

バリエーション
	•	サイズ
	•	カラー
	•	その他属性
	•	属性ごとの在庫数
	•	属性ごとのSKU

注文情報

	•	event_id
	•	participant_id または guest_id

	•	税額

注文状態

11. 状態遷移の設計ポイント

予約
	•	申込中 → 予約確定 → 来場済
	•	申込中 → 予約確定 → キャンセル
	•	申込中 → 定員超過で失敗

整理券
	•	発券済 → 待機中 → 呼出中 → 来場済
	•	発券済 → 待機中 → 呼出中 → 不在
	•	発券済 → キャンセル / 失効

注文
	•	カート相当 → 注文確定 → 準備中 → 呼出済 → 受取完了
	•	注文確定 → キャンセル

ここを甘く作ると、あとで現場が阿鼻叫喚になるので、状態遷移は enum で曖昧にせず、許可遷移を明示した方がいい。

13. バイブコーディング用実装指示文

以下は、そのままAI実装用プロンプトに流しやすい形。

イベント来場管理システムを構築してください。
目的は、イベント単位で「予約管理」「チェックイン」「整理券発券」「モバイルオーダー」を提供することです。

必須要件:
- イベントごとに機能を有効/無効化できる
- 来場者はWebまたはLINE導線からアクセスできる
- 予約機能は、フォーム入力、予約完了、前日通知、当日チェックインに対応する
- 整理券機能は、QRアクセス、発券、待機状況表示、管理画面からの呼び出し、通知、来場処理に対応する
- 物販注文機能は、商品一覧、カラー/サイズ選択、注文、スタッフ画面での確認、受取通知、受け渡し完了に対応する
- 管理画面で予約数、チェックイン数、整理券待機数、注文数、商品別販売実績を確認できる
- 通知テンプレートをイベントごとに設定できる
- 各機能は独立利用可能にする
- モバイルファーストで実装する
- リアルタイム更新は polling または WebSocket で実装する
- PostgreSQLを使い、主要テーブルは events, participants, reservations, queue_tickets, products, product_variants, orders, order_items, notifications とする
- ステータス遷移は明示的に管理する
- 在庫管理と呼び出し管理は同時更新競合を考慮する

画面:
- イベント公開ページ
- 予約フォーム
- 整理券発券ページ
- 整理券待機状況ページ
- 商品一覧/注文ページ
- 管理ダッシュボード
- 予約一覧
- チェックイン画面
- 整理券呼び出し画面
- 注文一覧/受け渡し画面
- 商品管理画面
- 通知テンプレート管理画面

まずはMVPとして、
1. イベント管理
2. 予約受付
3. チェックイン
4. 整理券発券と呼び出し
5. 商品注文と受取管理
6. ダッシュボード集計
を実装してください。
