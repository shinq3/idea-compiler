# IdeaCompiler — 機能詳細一覧

---

## 1. 認証・アクセス制御

| # | 機能 | 詳細 |
|---|------|------|
| 1-1 | JWT認証 | ユーザー名またはメールアドレス＋パスワードでログイン。JWTトークン（7日間有効）を発行し、以降のAPIリクエストに使用 |
| 1-2 | ロールベースアクセス制御（RBAC） | 4段階のロール：**system_admin**（全権限）、**org_admin**（組織管理）、**pm**（プロジェクト管理）、**member**（閲覧・入力のみ） |
| 1-3 | プロジェクト単位のアクセス制御 | system_admin＝全プロジェクト、org_admin/pm＝自組織のプロジェクト、member＝招待されたプロジェクトのみ閲覧・入力可能 |
| 1-4 | パスワードハッシュ化 | bcryptによるパスワードのハッシュ化保存（ソルトラウンド10） |

---

## 2. 組織管理

| # | 機能 | 詳細 |
|---|------|------|
| 2-1 | 組織の作成・編集・削除 | system_adminが組織を作成・削除。org_adminは自組織の名称を編集可能 |
| 2-2 | マルチテナント | プロジェクト・ユーザーは組織に紐付き、データが組織単位で分離 |
| 2-3 | 組織管理画面 | 組織一覧、メンバー数表示、スラッグ管理 |

---

## 3. ユーザー管理

| # | 機能 | 詳細 |
|---|------|------|
| 3-1 | ユーザーCRUD | ユーザーの作成・編集・削除。作成可能なロールは自分のロール以下に制限 |
| 3-2 | ロール別の管理権限 | system_admin＝全ユーザー管理、org_admin＝自組織のpm/member管理、pm＝memberの招待のみ |
| 3-3 | プロフィール編集 | 自分の表示名・メールアドレス・パスワードを変更可能 |
| 3-4 | ユーザー管理画面 | ユーザー一覧テーブル、作成・編集・削除ダイアログ付き |

---

## 4. プロジェクト管理

| # | 機能 | 詳細 |
|---|------|------|
| 4-1 | プロジェクト作成 | タイトルのみ、またはRFP（PDF）アップロード付きで作成。予算レンジ・リリース日・担当者を設定可能 |
| 4-2 | ダッシュボード | プロジェクト一覧表示。ステータス（discovery/proposal/negotiation/won/lost）、組織、キーワードでフィルタリング |
| 4-3 | プロジェクト詳細画面 | タブ切替で入力・抽出データ・サマリー・ドキュメント・メンバーを管理 |
| 4-4 | ステータス管理 | プロジェクトのライフサイクルステータスを変更可能 |
| 4-5 | プロジェクト削除 | 確認ダイアログ付き。関連データ（入力・抽出データ・サマリー・ドキュメント）もカスケード削除 |
| 4-6 | プロジェクトメンバー管理 | pm/org_admin/system_adminがメンバーを追加・削除。Viewer/Editorの権限設定 |

---

## 5. AI入力・分析

| # | 機能 | 詳細 |
|---|------|------|
| 5-1 | テキスト入力 | フリーテキストで議事録・メモ・要件を入力 |
| 5-2 | PDF/ファイルアップロード | PDF、TXT、MDファイルをアップロード。PDFはpdf-parseでテキスト抽出 |
| 5-3 | 音声入力（Whisper） | ブラウザのマイクで録音 → OpenAI Whisper（whisper-1）でリアルタイム文字起こし → テキストとして入力 |
| 5-4 | AI構造化データ抽出 | 入力テキストからAI（o4-mini）が自動分類・抽出。カテゴリ：要件（Must/Nice-to-have）、課題、決定事項、制約、予算、スケジュール、リスク、用語、アクション（質問） |
| 5-5 | 3言語同時翻訳 | 入力テキスト・プロジェクト名・顧客名・抽出項目を日本語・英語・ベトナム語に自動翻訳 |
| 5-6 | 入力履歴管理 | 過去の入力を一覧表示、編集・削除可能。編集時はAI再抽出・サマリー再生成をトリガー |
| 5-7 | PDF再アップロード | PDF抽出失敗時に再アップロード可能（デプロイ再起動でファイル消失した場合の対応） |

---

## 6. AI自動サマリー

| # | 機能 | 詳細 |
|---|------|------|
| 6-1 | プロジェクトサマリー自動生成 | 入力追加のたびにAIが総合サマリーを自動生成・更新 |
| 6-2 | サマリーのバージョン管理 | サマリーは入力のたびにバージョニング。過去バージョンを一覧・比較可能 |
| 6-3 | サマリー構成 | 概要、課題、スコープ、機能候補、予算、スケジュール、リスク、未解決事項など |
| 6-4 | 信頼度ゲージ | 予算・スケジュール・要件の情報充足度をAIが0〜100%で評価し、ビジュアルゲージで表示 |
| 6-5 | 3言語サマリー | 日本語・英語・ベトナム語のサマリーを同時生成 |

---

## 7. ドキュメント生成

| # | 機能 | 詳細 |
|---|------|------|
| 7-1 | キックオフドキュメント生成 | AIがプロジェクト情報からキックオフ資料（Markdown）を自動生成。背景・目的、課題、ゴール、スコープ、体制、マイルストーン、リスク、次のアクション |
| 7-2 | 機能提案書生成 | 抽出された要件をもとにAIが機能提案書を自動生成 |
| 7-3 | 3言語ドキュメント | ドキュメントは日本語・英語・ベトナム語を同時生成 |

---

## 8. スライド生成

| # | 機能 | 詳細 |
|---|------|------|
| 8-1 | AIスライド自動生成 | キックオフドキュメント・機能提案書をreveal.jsスライドに変換。タイトル、セクション区切り、コンテンツ、KPI、タイムライン、リスク等のスライドタイプに対応 |
| 8-2 | アプリ内プレビュー | 生成されたスライドをアプリ内のiframeでリアルタイムプレビュー |
| 8-3 | HTMLダウンロード | reveal.jsライブラリを内蔵したスタンドアロンHTMLファイルとしてダウンロード可能。オフラインでプレゼン可能 |

---

## 9. ランディングページ（公開）

| # | 機能 | 詳細 |
|---|------|------|
| 9-1 | サービス紹介ヒーロー | 背景画像付きヒーローセクション。サービスのキャッチコピーとサインアップ導線 |
| 9-2 | 8シーンストーリーボード | プロジェクトの課題→解決をストーリー仕立てで紹介。AIイラスト付き、左右交互レイアウト |
| 9-3 | ライブAIデモ | ログイン不要でAI分析を体験可能。テキスト入力・ファイルアップロード・音声録音に対応。サマリーと構造化データをリアルタイム表示 |
| 9-4 | デモログ保存 | デモ実行結果は「LP Demo」組織のプロジェクトとしてDBに保存 |
| 9-5 | 注意書き表示 | 機密情報の入力禁止を赤字で注意喚起 |

## 10. 多言語対応（i18n）

| # | 機能 | 詳細 |
|---|------|------|
| 10-1 | UIの3言語切替 | 日本語（デフォルト）・英語・ベトナム語。ヘッダーの言語切替プルダウンで即時切替 |
| 10-2 | 言語設定の永続化 | 選択した言語はlocalStorageに保存し、次回アクセス時も維持 |
| 10-3 | AIコンテンツの3言語同時出力 | 抽出項目・サマリー・ドキュメントは3言語同時生成。UI言語に合わせて自動表示 |

## 11. PWA（Progressive Web App）

| # | 機能 | 詳細 |
|---|------|------|
| 11-1 | インストール可能 | ホーム画面に追加でネイティブアプリのように利用可能（iOS/Android/デスクトップ） |
| 11-2 | オフラインキャッシュ | Service Workerによりコアアセットをキャッシュ。オフラインでも基本画面を表示 |
| 11-3 | Webアプリマニフェスト | アイコン・テーマカラー・スプラッシュ画面を設定 |

## 12. 技術基盤

| # | 項目 | 技術 |
|---|------|------|
| 12-1 | フロントエンド | React + TypeScript + Vite + TailwindCSS + shadcn/ui |
| 12-2 | バックエンド | Express.js（Node.js / TypeScript） |
| 12-3 | データベース | PostgreSQL（Drizzle ORM） |
| 12-4 | AIエンジン | OpenAI API（o4-mini / gpt-4o-mini / Whisper-1） |
| 12-5 | 認証 | JWT（jsonwebtoken + bcrypt） |
| 12-6 | ルーティング | wouter（フロントエンド） |
| 12-7 | データフェッチ | TanStack Query v5 |
| 12-8 | ファイル処理 | multer（アップロード）+ pdf-parse（PDF抽出） |
| 12-9 | スライド | reveal.js（HTML生成＋スタンドアロンエクスポート） |
| 12-10 | ホスティング | Replit Deployments |

  [自動サマリー生成（バージョニング）]
        ↓
  [信頼度ゲージ更新]
        ↓
  [キックオフドキュメント / 機能提案書 生成]
